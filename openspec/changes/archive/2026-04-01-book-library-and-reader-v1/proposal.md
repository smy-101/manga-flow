## Why

manga-flow 目前只有 Tauri 脚手架，没有任何业务功能。V1 的目标是实现最小可用的漫画阅读体验：用户能够导入本地漫画文件，在书库中浏览，点进去翻页阅读。这是整个产品的基础——没有书库和阅读器，翻译、排版等增强功能无处承载。

## What Changes

- 新增基础设施：React Router v7（库模式）前端路由、Zustand 状态管理、基础布局框架（侧栏导航 + 主内容区）
- 新增 Rust 后端文件扫描层：支持文件夹、zip、cbz 三种格式的漫画导入，统一解压/复制为内部文件夹结构
- 新增 SQLite 数据库：通过 tauri-plugin-sql 管理书籍、章节、页面元数据及阅读进度
- 新增书库管理页面：封面网格展示、导入对话框（选择文件夹/文件）、删除书籍
- 新增阅读器页面：单页翻页浏览、键盘快捷键（← →）翻页、阅读进度自动持久化
- 新增首次使用引导：选择漫画库存放目录，设置中可修改
- 导入时将漫画文件复制到用户选择的库目录，统一归一化为 `library/books/{id}/pages/` 文件夹结构

## Capabilities

### New Capabilities

- `library-setup`: 漫画库目录初始化与配置（首次使用引导、目录选择、存储结构）
- `book-import`: 漫画文件导入（支持文件夹/zip/cbz、格式检测、解压复制、元数据提取、封面缩略图生成）
- `book-storage`: 书库数据管理（SQLite 建表、书籍/章节/页面 CRUD、数据模型）
- `library-ui`: 书库界面（封面网格、导入操作、删除、最近阅读、阅读进度显示）
- `reader`: 漫画阅读器（单页翻页、键盘快捷键、图片加载、阅读进度持久化与续读）

### Modified Capabilities

（无已有能力需要修改）

## Impact

- **前端依赖新增**：react-router v7、zustand、@tauri-apps/plugin-sql
- **Rust 后端新增依赖**：zip crate（zip/cbz 解压）、tauri-plugin-sql
- **Tauri 配置变更**：注册 sql 插件、配置文件系统权限（读写用户选择的库目录、app_data_dir）
- **磁盘空间**：导入漫画会创建副本，需要用户有足够磁盘空间
- **新增路由**：`/`（书库）、`/reader/:bookId`（阅读器）、`/settings`（设置）

## 非目标

- 双页模式、连续滚动模式（V2）
- 阅读进度百分比角标、已读完标记（V2 细化）
- rar 格式支持（V2）
- 漫画翻译功能（V2）
- 智能章节解析、元数据自动识别（V3）
- 标签、收藏、搜索等高级书库管理（V2）
- 图片格式转换或压缩
- 磁盘剩余空间提示

## 测试策略

- **前端测试**：Vitest + React Testing Library

  - 用户交互测试（导入流程、翻页、键盘事件）
  - 状态管理测试（Zustand store）
- **Rust 后端测试**：Cargo 内置测试框架
  - 文件扫描逻辑（文件夹、zip、cbz 格式解析）
  - 数据模型单元测试
- **集成测试**：
  - 导入 → 书库显示 → 点击阅读 完整流程
  - 阅读进度保存与续读
